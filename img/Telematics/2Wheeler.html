VIDA VCI

USB-based VCI Tool for Two Wheeler Electric Vehicles

We have designed these Diagnostic Tools which have two components. The first one is a Hardware (VCI Tool) that connects to the Vehicle's OBD Port. The second component of the Diagnostic system is a use-friendly Software/Application, that is installed on a Desktop/Laptop. A USB Cable connects the VCI tool to the Computer. The User operates the VCI tool from the Desktop Application.We provide customized tools and the user applications as per our clients requirements, regarding the Looks, functionalities and compatibility with the ECUs used by them.

Both the standard OBD-II Protocols and OEM Specific Diagnostic Functions are supported. The user can use it for Diagnostic Functions, such as checking Diagnostic Trouble Codes (DTCs), Live Parameters (e.g., Engine RPM, Air In-take Temperature, Oxygen Levels, Exhaust Temperatures, etc.)

Our FAQ Frequently Asked Questions
Do I need to check the battery voltage before start the flashing process ?

Yes, battery voltage should be more than 11V before start the flashing.

What if the vehicle battery is less than 11V and I try to flash the vehicle?

Flashing may fail if battery voltage is less than 11V, otherwise it may fail during the process. It is recommended to have a fully charged battery.

While setting flash file path, do I need to run VCI application as “Run as administrator”?

Yes, set flashing file path must be done in administrator mode. Also remove the existing “DATASET” flashing file folder.

Do I need to set flashing file path every time I do online / offline flashing?

No. you have to set flashing file path first time only in administrator mode.

If flashing is failed, do I have to re-flash the vehicle using specific mode (online/offline)?

No, vehicle can be re-flashed using any of the mode –Offline/Online.

If flashing is failed in one model, is it advisable to shift to another model and do the flashing?

No, user should re-flash the same model before shifting to the next model for flashing.

If flashing is failed in one model, can I shift to another model to diagnose it?

Yes, you can diagnose another model and can later comeback to re-flash previous model.

What are the possible reasons for Flashing Failure?

1. VCI application / VCI tool version not up to date
2. Vehicle battery voltage less than 11V
3. Vehicle ignition off during ECU flashing
4. Poor internet connection (in online flashing only)
5. Loose Connection between OBD cable and DLC Coupler
6. Loose connection between VCI tool and USB cable
7. By pressing reset button in VCI application
8. By pressing reset button in VCI tool

TVS End of Line

USB CDC VCI Tool for EOl Application for Two Wheeler Vehicle

The VCI tool is a wired tool. The VCI tool connects to a computer (pc, laptop, industrial computer) via a usb cable. The tool is powered from the usb power of the host computer, and hence is isolated from the power surges in the vehicle power circuit. A fast, responsive, user-friendly desktop application developed natively in dot net c# is used as the user interface to display the ECU diagnostic and flashing information. The software is compatible with windows computers. Being a wired tool, the communication between the VCI tool and the monitoring station remains more reliable. The VCI tool has internal memory, which makes it easy to store the reusable files for fast operation in the factory assembly line, without having to resort to transferring the ECU firmware file from pc to VCI tool everytime. This improves the performance of the vehicle ECU flashing process significantly.

VCI E-lite

Android-Based VCI tools

These Diagnostic Tools have two components. The first one is a Hardware (VCI Tool) that needs to be connected to the Vehicle's OBD Port. The Second part of the Diagnostic system is a Use-Friendly Application (Android App), which can be downloaded from the Google Play store and Installed onto an Android Smart Phone. The Android Phone communicates with the Hardware VCI Tool wirelessly over Bluetooth. The User operates the VCI tool from the Android Application.

Both the standard OBD-II Protocols and OEM Specific Diagnostic Functions are supported. The user can use it for Diagnostic Functions, such as checking Diagnostic Trouble Codes (DTCs), Live Parameters (e.g., Engine RPM, Air In-take Temperature, Oxygen Levels, Exhaust Temperatures, etc.)
